# Thymus-Derived Myeloid Education Signatures (MES)

**Thymus-derived myeloid programs track microglial tolerance states across human cohorts**

Sanwal Ahmad Zafar, Wei Qin\*, Liu Chengliang, Areeba Ali Khan, Muhammad Salman Faisal

\*Corresponding author: Prof. Wei Qin (wqin@sjtu.edu.cn)

---

## Overview

This repository contains the complete analysis pipeline for deriving thymus-based Myeloid Education Signatures (MES) and evaluating their relationship with microglial tolerance positioning across independent human cohorts.

We show that:

- The human thymus encodes eight conserved MES modules (chemokine, ECM/adhesion, axon guidance, and tolerance-associated axes)
- These programs are inversely associated with microglial functional positioning when transferred to four independent brain cohorts (SEA-AD, Olah, Tuddenham, MS)
- Glucocorticoid receptor (GR) activity does **not** moderate MES–tolerance associations under a formal per-module interaction model (well-powered null)
- Findings replicate in Visium spatial transcriptomics and LPS-tolerized microglia

## Setup

### Data

Raw data are publicly available from:

- **Tabula Sapiens Thymus and Blood** — tabula-sapiens-portal.ds.czbiohub.org and CELLxGENE
- **SEA-AD microglia + immune** — portal.brain-map.org/explore/seattle-alzheimers-disease
- **Olah et al. live human microglia** — CELLxGENE
- **GEO datasets** — GSE144870 (Lavaert thymus), GSE139042 (Le thymus), GSE133341 (Zeng multi-tissue), GSE204702 (Tuddenham cross-disease microglia), GSE138852 (Mathys AD snRNA), GSE180759 (Absinta MS lesion), GSE219208 (microglia GR perturbation), GSE229940 (innate memory bulk), GSE184241 (innate memory LPS), GSE220442 (Visium), GSE233208 (paired Visium + snRNA), GSE135618 (sc vs sn QC)

See Supplementary Table S1 (generated by NB1) for the full dataset catalog with accession numbers, sample sizes, and access dates.

### Running

1. Clone this repository
2. Download raw data into your local project directory, following the directory layout encoded in NB1's dataset registry. The expected top-level structure is `<project_root>/Raw Data/{Thymus, Brain, Spatial transcriptomics, ...}/`
3. Set `MES_BASE_DIR` to point at your project root, or run from a directory that already contains the data folders:

```bash
export MES_BASE_DIR=/path/to/your/project
```

4. Run notebooks sequentially (NB1 → NB8) in Jupyter:

```bash
git clone https://github.com/Sjtu-Fuxilab/innate.git
cd innate
export MES_BASE_DIR=/path/to/your/project
jupyter notebook
```

NB1 will fail at its `assert RAW_DIR.exists()` check if the data path is not set up correctly; downstream notebooks (NB2 onwards) consume processed AnnData objects from `<MES_BASE_DIR>/Process Data/` produced by earlier notebooks. NB7 and NB8 depend on NB3–NB5 outputs and can be run independently once those are complete.
